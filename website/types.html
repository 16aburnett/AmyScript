AmyScript has a few built-in types. Currently there isn't a system to create sub-types or unions, however, you can create your own classes which become a new Object type.

Integers (int)

Integers are any number (positive or negative) without a decimal point. This is one of the most common types that you could use all over like in loops, conditionals, data, etc.

  • 0
  • 1
  • 12131421313
  • -18
  • -594

Unbounded: Fortunately, the AmyAssembly interpreter is written in Python3 which means any integer has an unbounded range! This means you do not have to worry about overflow which is a common concern in languages like C, C++, or Java.

You can declare an integer variable using the below statement. This is very similar to C++ and Java. Integer values can be expressed by simply typing the number (-7), however, it cannot have a decimal point!

Integer Variable

int x = -7;

Floating Point Numbers (float)

Floating point numbers are any number that includes a decimal point. Here are some examples of valid floating point numbers:

  • 1.0
  • 3.14159
  • -6.25
  • 0.005
  • 0.0

Floating Point Variable

float y = 3.14;

Characters (char)

Characters are letters, numbers, whitespace, symbols, etc.

Character literals are denoted by single quotes surrounding a single character. Examples:

  • 'a'
  • '9'
  • ' '
  • '\n'

Character Variable

char c = 'w';

Void Types (void)

Void types are a little weird. They are most commonly used with functions to describe that the function does not return anything.

Void Type

function void myFunction () {...}

NOTE: Variables cannot have a void type.

Arrays

Arrays are an incredibly useful data type. An array in AmyScript is a homogeneous list of values. You can have an array of integers, floats, characters, or even other arrays.

Below is an array of 5 integers.

Array

int[] nums = new int[5];

The nums variable contains space for the 5 integers and you can use nums to access those values.

The Array section describes arrays in more detail.

String Literals

AmyScript does not have a string type for storing a list of characters, however, you can always create an array of characters. For convenience, AmyScript allows you to easily create character arrays for strings of text using string literals.

A string literal is denoted using double quotes (similar to other languages). See below:

String Literals

char[] str = "This is a string of text!";

Objects

An object is another data structure that can store more than one element. However, an object stores specific values and has functions that operate on that object's data.

Objects are a part of Object-Oriented Programming and exist within C++ and Java. AmyScript follows a similar syntax to C++ and Java for declaring and creating objects.

Below is a brief example of creating an object.

Object

// Definition for an object/class
// this describes the structure of the object
class Person
{
    public field char[] name;
    public field int    age; 
    constructor (char[] name, int age)
    {
        this.name = name;
        this.age  = age;
    }
}

// Creating an actual instance of the object
Person person = new Person("Amy", 22);

The person variable contains an object that is structured like the class definition for a Person. The person object stores 2 elements: a name (char[]) and an age (int).

The Classes section describes objects in more detail.
