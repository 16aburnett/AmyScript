AmyScript - Operators

Operators are symbols that represent a mathematical, relational, or logical operation.

AmyScript has many different operators that do different things. Below are the different types of Operators.

  • Arithmetic Operators
  • Assignment Operators
  • Comparison Operators
  • Logical Operators
  • Other Operators
  • Operator Precedence

Arithmetic Operators

Operator Description Example
+ Addition
Adds LHS and RHS		 
x = 3 + 7;
- Subtraction/Negation
Subtracts RHS from LHS
Flips the sign of a value 
x = 10 - 3;
x = -y;
* Multiplication
Multiplies LHS by RHS		 
x = 2 * 5;
/ Division
Divides LHS from RHS		 
x = 25 / 5;
% Modulus (Division Remainder)
Divides LHS by RHS and returns remainder		 
x = 9 % 2;
++ Increment*
Pre-increment (++x) adds 1 to x and returns the new value.
Post-increment (x++) adds 1 to x and returns the old value. 
x = y++;
x = ++y;
-- Decrement*
Pre-decrement (--x) subtracts 1 from x and returns the new value.
Post-decrement (x--) subtracts 1 from x and returns the old value. 
x = y--;
x = --y;
() Parentheses
Gives an expression higher precedence 
x = (3 + 5) * 10;

* These operators perform an assignment and therefore require a variable.

Comparison Operators

AmyScript does not have a boolean type (true/false), therefore, AmyScript uses integers to represent true or false. Any non-zero value represents true, and zero represents false.

Comparison operators are useful for If-Statements and Loops.

Operator Description Example
== Equal to
Returns true if LHS equals RHS, false otherwise 
x = 5 == 5;// true
y = 6 == x;
// false, x is 5 which doesnt equal 6
!= Not equal to
Returns true if LHS does not equal RHS, false otherwise 
x = 5 != 5;// false
y = 6 != x;
// true, x is 5 which doesnt equal 6
< Less than
Returns true if LHS is less than RHS, false otherwise 
x = 3 < 5;//true
x = 5 < 3;//false
<= Less than or equal to
Returns true if LHS is less than or equal to RHS, false otherwise 
x = 3 <= 5;//true
x = 3 <= 3;//true
x = 5 <= 3;//false
> Greater than
Returns true if LHS is greater than RHS, false otherwise 
x = 5 < 3;//true
x = 3 < 5;//false
>= Greater than or equal to
Returns true if LHS is greater than or equal to RHS, false otherwise 
x = 5 >= 3;//true
x = 3 >= 3;//true
x = 3 >= 5;//false

Logical Operators

Operator Description Example
&& Logical AND
Returns true if LHS and RHS are true, false otherwise 
x = 1;
y = 0;
x && x //true
x && y //false
y && x //false
y && y //false
|| Logical OR
Returns true if LHS or RHS is true, false otherwise 
x = 1;
y = 0;
x || x //true
x || y //true
y || x //true
y || y //false
! Logical NOT
Returns true if operand is false, false otherwise 
x = 1;
y = 0;
!x //false
!y //true

NOTE: AmyScript does not yet have short circuit evaluation which is a feature in other languages like C, C++, and Java. This means that if the LHS of Logical AND is false, then the RHS will still be evaluated.

Other Operators

Operator Description Example
[] Subscript
Accesses a specific index from an array. 
int[] nums = new int[5];
nums[0] = 17; //Assignment
nums[1] = nums[0] * 2; //Grabs value
() Function Call
Returns the value returned by the function 
function int add (int a, int b)
{
   return a + b;
}
add (5, 7); // returns 12
. Dot Operator (Member Accessor)
Accesses a member of a class (field or method) 
class Person
{
    public field char[] name;
    private field int   age;
    constructor (char[] name, int age)
    {
        this.name = name;
        this.age = age;
    }
    public method int getAge ()
    {
        return this.age;
    }
}
Person person = new Person ("Amy", 22);
person.name;      // returns "Amy"
person.getAge();  // returns 22 (int)

Operator Precedence

In expressions, some operators are evaluated before others which is called Operator Precedence. AmyScript uses a very similar operator precedence to C++ (C++ Operator Precedence). Below is the precedence chart. Operators at the top are evaluated first starting with paretheses similarly to in mathematics (Order of Operations).

Associativity is the order that successive operators of the same precedence are evaluated. For example, subtraction has left to right associativity since in mathematics we evaluate it left to right (3 - 1 - 2 = 2 - 2 = 0), if it were right to left, then we would get a different result (3 - 1 - 2 = 3 - -1 = 4).
